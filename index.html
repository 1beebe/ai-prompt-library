As You Go: Notes for Learners

  • Pay attention to when your AI assistant gives you exactly what you need — that's a prompt worth saving. Copy it somewhere.
  • Start building your own prompt library: keep a simple doc, label what works, and share it with your team.
  • You can ask your AI assistant how to prompt better: "How should I phrase this to get a clearer answer?" — it'll tell you.
  • It's okay to go slow. One new prompt per week is still a real skill. You don't have to become an expert overnight.
  • Treat it like a conversation, not a search box: follow up, push back, ask for more. The second message is often better than the first.
  • If an answer feels off, say so: "That's not quite right — can you try again with more focus on X?" Your AI assistant doesn't get offended.

How This Was Built

Input

A project brief built from stakeholder interviews and business requirements gathering. The goal: give every department copy-paste-ready AI prompts they could actually use — not a demo, not a slideshow, but a working tool. The format is inspired by OpenAI's Academy prompt packs for different functional teams.

Build

The brief went to Claude Code, which generated a single self-contained HTML file — filters, copy buttons, cards, and all — through iterative prompting and feedback loops. No front-end framework. No dependencies beyond a CDN icon set.

Quality Check

Output was reviewed by prompting Claude Code to evaluate the file from the perspective of a CEO, a COO, and a UX reviewer. That surfaced structural changes like moving Thought Partner to the top, adding card-level filtering, and fixing anchor links. Manual QA and additional iterations followed.
